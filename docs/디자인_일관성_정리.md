# 대시보드·영수증 디자인 일관성 정리

## 1. 분석 결과 (변경 전)

| 구분 | 대시보드 | 영수증 목록 | 네비게이션 | 앱 공통 |
|------|----------|-------------|------------|--------|
| **배경** | 흰색 카드 (#fff) | 밝은 회색 페이지 (#f5f5f5) | 흰색 (#fff) | 다크 (#1a1a1a) |
| **강조색** | #2196F3 | #2196F3, 삭제 #ff5722 | #2196F3 | #4a90d9, 위험 #e74c3c |
| **테마** | 라이트 고정 | 라이트 고정 | 라이트 고정 | 다크 + 라이트(시스템) |

- **문제**: 앱은 다크/라이트를 `:root` 변수로 쓰는데, 대시보드·영수증은 라이트 색상을 하드코딩해 **페이지마다 톤이 달랐음**.
- **색 불일치**: Primary가 App은 `#4a90d9`, 대시/영수증/네비는 `#2196F3`. 삭제/초기화는 App `#e74c3c`, 영수증 `#ff5722`.

---

## 2. 적용한 디자인 시스템 (변경 후)

모든 페이지가 **App.css의 `:root` 변수**만 사용하도록 통일했습니다.

| 변수 | 용도 |
|------|------|
| `--color-bg` | 페이지·입력 필드 배경 |
| `--color-bg-secondary` | 카드·섹션·모달 배경 |
| `--color-bg-tertiary` | 구분선, 비활성 배경, 테두리 |
| `--color-primary` | 버튼(퀵버튼·날짜배지·순위뱃지·금액 강조·active 네비) |
| `--color-danger` | 삭제·필터 초기화 버튼 |
| `--color-text` | 본문·제목 |
| `--color-text-muted` | 보조 문구·라벨 |
| `--spacing-xs/sm/md/lg` | 여백·패딩 |
| `--font-size-xs/sm/md/lg/xl` | 글자 크기 |
| `--border-radius` | 모서리 (8px) |
| `--nav-height` | 하단 네비 높이 |

---

## 3. 변경 요약

### Dashboard.css
- 배경/카드/입력: `#fff` → `var(--color-bg-secondary)`, `var(--color-bg)`
- 텍스트: `#333`/`#666` → `var(--color-text)`, `var(--color-text-muted)`
- 강조/버튼: `#2196F3` → `var(--color-primary)`
- 여백/폰트: 고정값 → `var(--spacing-*)`, `var(--font-size-*)`
- 테두리: `#ddd`/`#eee` → `var(--color-bg-tertiary)` 또는 1px solid
- 박스 섀도우 제거, 테두리로 구분 (다크에서 자연스럽게)

### ReceiptList.css
- 페이지/필터/카드/모달: 위와 동일하게 변수 적용
- 삭제·초기화: `#ff5722` → `var(--color-danger)`
- 날짜 배지·포커스: `var(--color-primary)`

### Navigation.css
- 배경: `#fff` → `var(--color-bg-secondary)`
- 테두리: `var(--color-bg-tertiary)`
- 텍스트/활성: `var(--color-text-muted)`, `var(--color-primary)`
- 패딩/높이: `var(--spacing-*)`, `var(--nav-height)`

---

## 4. 결과

- **다크 모드**: 헤더·메인·대시보드·영수증·네비가 같은 다크 톤으로 통일.
- **라이트 모드**: 시스템 설정이 라이트일 때 App.css의 `prefers-color-scheme: light`로 `:root`가 바뀌므로, 대시보드·영수증·네비도 자동으로 라이트로 표시.
- **색·간격·타이포**: Primary/ Danger 한 가지씩, 간격·폰트는 변수로 통일해 일관된 레이아웃과 스타일 유지.

이제 대시보드·영수증 레이아웃과 스타일이 앱 전체와 일관된 디자인으로 구성되어 있습니다.
