# Render 무료 플랜 제한 사항 (개인 사용 시)

참고: [Render 공식 문서 – Deploy for Free](https://render.com/docs/free)

---

## 이 프로젝트 구성 (기준)

- **백엔드**: Render Web Service (Free)
- **프론트엔드**: Render Static Site (Free)
- **DB**: Supabase (Render Postgres 아님 → 아래 “Free Postgres” 제한은 해당 없음)

---

## 1. 백엔드(Web Service) 무료 플랜 제한

### 슬립(Spinning down)
- **15분 동안 요청이 없으면** 서비스가 자동으로 **슬립(중지)** 됩니다.
- 다음 요청이 오면 다시 켜지는데, **켜질 때까지 최대 약 1분** 걸릴 수 있습니다.
- **개인용**: 가끔 쓰면 첫 접속·첫 API 호출이 30초~1분 정도 느리게 느껴질 수 있습니다.

### 월 사용 시간 (750시간)
- **Free instance hours**: 워크스페이스당 **월 750시간**까지 무료.
- 슬립 상태일 때는 시간을 **소모하지 않습니다**. 켜져 있을 때만 소모.
- 750시간 = 한 달을 24시간 켜두면 약 31일 분량이므로, **백엔드 1개만 쓸 때는 보통 다 쓸 일이 거의 없습니다** (슬립 때문에 실제로는 훨씬 적게 씀).
- 750시간을 다 쓰면 **그 달 남은 기간 동안 Free 웹 서비스가 전부 중지**되고, 다음 달 1일부터 다시 750시간으로 리셋됩니다.

### 그밖에
- **로컬 디스크**: 재배포·재시작·슬립 시 **파일이 다 사라짐** (영구 디스크 없음). 이 프로젝트는 DB를 Supabase에 두므로 문제 없음.
- **인스턴스 1개만** 사용 가능 (스케일 아웃 불가).
- Render가 **아무 때나 재시작**할 수 있음 (가끔 짧은 다운 가능).

---

## 2. 프론트엔드(Static Site) 무료 플랜

- Static Site는 **슬립 없음** (항상 켜져 있는 형태).
- **트래픽·빌드**는 월간 “included” 한도(대역폭, pipeline 분) 안에서 사용.
- 개인용으로 쓰면 **일반적으로 한도 걸릴 가능성 낮음**.

---

## 3. 대역폭·빌드 (공통)

- **아웃바운드 대역폭** / **빌드 파이프라인 분**은 월간 포함량이 있고, 초과 시:
  - 결제 수단이 있으면: 초과분 과금.
  - 결제 수단이 없으면: **서비스가 그달 남은 기간 동안 중지**될 수 있음.
- 사용량은 [Billing 페이지 – Monthly Included Usage](https://dashboard.render.com/billing#included-usage)에서 확인 가능.

---

## 4. 이 프로젝트에서 개인용으로 쓸 때 정리

| 항목 | 내용 |
|------|------|
| **비용** | Free 플랜은 **무료**. (대역폭/빌드 초과 시만 과금 또는 중지) |
| **슬립** | 백엔드만 15분 미사용 시 슬립 → **첫 요청이 30초~1분 느릴 수 있음**. |
| **월 750시간** | 백엔드 1개만 쓰고 슬립 잘 되면 **개인용으로는 보통 여유** 있음. |
| **DB** | Supabase 사용 시 Render Postgres 30일 만료/1GB 제한은 **해당 없음**. |
| **용도** | Render는 Free를 **프로덕션용으로 권장하지 않음**. 테스트·취미·개인용에는 적합. |

---

## 5. 개인용으로 불편할 수 있는 점 요약

1. **첫 사용이 가끔 느림** → 15분 이상 안 쓰면 백엔드 슬립 → 다음 접속 시 웨이크업 대기.
2. **월 750시간 초과** → 그달 남은 기간 백엔드 중지 (개인용으로는 드묾).
3. **대역폭/빌드 초과** → 결제 없으면 서비스 중지 가능 (사용량 적으면 거의 해당 없음).
